JavaScript Getters and Setters

  JavaScript can secure better data quality when using getters and setters.
  Using the lang property, in this example, returns the value of the language property in upper case:
  Getters and setters allow you to get and set properties via methods.
Data Quality Getters 
    <script>
        // Create an object:
        const person = {
        firstName : "Deli",
        lastName  : "Dolu",
        language  : "en",
        get lang()  {
          return this.language.toUpperCase();
        }
      };
      // Display data from the object using a getter:
      document.getElementById("demo").innerHTML = person.lang;
      </script>
      <p id="demo"></p>

Data Quality Setters 
  <script>
    // Create an object:
      const persn  = {
      firstName : "Deli",
      lastName  : "Dolu",
      language  : "",
      set lang(lang)  {
        this.language = lang.toUpperCase();
      }
    };
    
    // Set a property using set:
    persn.lang = "en";
    
    // Display data from the object:
    document.getElementById("dem").innerHTML =persn.language;
    </script>
    <p id="dem"></p>

Answer : 

   Why Using Getters and Setters?
  # It gives simpler syntax
  # It allows equal syntax for properties and methods
  # It can secure better data quality
  # It is useful for doing things behind-the-scenes
